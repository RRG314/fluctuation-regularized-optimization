# v0.1.0 - Initial Public Research Release

This is the first public release of `fluctuation-regularized-optimization`,
an MIT-licensed research-code repository for three optimization mechanisms:

- `LifshitzSwarm`: derivative-free black-box search with fitness-graded
  Lifshitz-style coupling and partial final quench refinement.
- `ZeroPointOptimizer`: PyTorch optimizer using zero-point-smoothed
  antithetic gradient estimates.
- `GradientPressureBalancer`: adaptive pressure balancing for PINNs and other
  multi-term losses.

## What Is Included

- Source-installable Python package under the `fluctuation_opt` import name.
- MIT license and citation metadata.
- Unit and integration tests.
- Quickstart example.
- Reproducible benchmark scripts and checked-in benchmark outputs.
- Mechanism/math documentation, API notes, reproducibility guide, roadmap,
  changelog, and contribution/security/support docs.
- Manuscript source in `paper/paper.tex`.

## Validation Summary

The current release state has been validated with:

- `pytest -q`: 89 passing tests.
- `python examples/quickstart.py`: passes.
- `python -m build --wheel`: builds a wheel containing `fluctuation_opt`.
- `python -m pip check`: no broken requirements.
- GitHub Actions test workflow.

Headline benchmark results are summarized in `REPORT.md`; raw CSV/JSON results
and figures are in `benchmarks/results/`.

## Important Caveats

- This is research code, not a universal optimizer superiority claim.
- The package is not published on PyPI; install from source.
- Smooth benchmark functions still favor tuned PSO or differential evolution
  for final precision.
- Heat-equation and housing-regression results show a robustness/accuracy
  tradeoff rather than a free win.
- The real Casimir data fit pins the separation offset at the configured lower
  bound and should be treated as a caveat.
- A compiled paper PDF is not attached because the current machine does not
  have a LaTeX engine available to regenerate it from source.
